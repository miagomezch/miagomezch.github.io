GIT

Introduccion

Git es un sistema de control de versiones, permite llevar el control de los cambios realizados en un proyecto, tambien puede encontrar la diferencias entre las versiones y trabajar en equipo.

  • Creado por Linus Torvalds in 2005
  • Usos:
    • Coordinar trabajo
    • Journal - Historial de las modificaciones
  • Comandos
    • git config --global user.name "nombre": establece el nombre de usuario, importante para la confirmacion de commit.
    • git config --global user.email "email": establece un correo electronico.
    • git init (nombre): crea un nuevo repositorio.
    • git status: muesta el estado del repositorio.
    • git add (nombre archivo): Se encarga de decirle al git que agrege los cambios para posteriormente hacer un commit.
    • git diff: muesta la diferencia enter los cambios de diferentes commits en el repositorio.
    • git checkout -- (nombre archivo): revierte los cambios de los cambios agregados para el commit.
    • git git reset --hard: Elimina completamente un commit.
    • git log: Permite ver los diferentes commits realizados.
    • git pull: recibe datos del repositorio.
    • git push: manda datos al repositorio.
    • git merge: une las ramas a la rama maestra.
