Rest

Rest

  • Estilo de arquitectura
  • Basado en recursos en contraste de basado en acciones
  • Apache
    • Contenido estático de una página web
  • Http
    • Es un protocolo de aplicación
    • En 2018 se promovió protocolo http 3
    • Métodos para crear el CRUD
      • GET
        • Es una petición al servidor o recurso por ser REST
        • Debe ser idempotente
        • El get se escribe en el barra de direcciones.
          • URI - URL
          • Debe ser idempotente
          • Cada vez que escribo una dirección estoy haciendo un GET
      • HEAD
      • POST
        • Para crear Get
        • No es idempotente
        • Summit a un formulario le estoy haciendo un POST
        • Enviar una contraseña esta dentro del POST
      • PUT
        • Actualizar en bloque
      • PATCH
      • DELETE
        • Si es idempotente
        • Si se borra en la base de datos, cuando lo vuelva a borrar sale error por lo que no es idempotente, pero para volverlo idempotente se marca como anulada la tabla, no se elimina la información, una tra
      • CONECT
      • OPTIONS
      • TRACE
    • Si la función está cambiando, está mal implementada
  • Tiene seis restricciones, uno opcional, los otros obligatorios
    • Uniform interface
    • Stateless
    • Cacheable
    • Cliente-servidor
    • Sistema en capas
    • Código por demanda

Resultados

  • Aplicaciones que cumplan con las 5 restricciones cumple beneficios como
    • Portabilidad
    • Escalabilidad
    • Confiabilidad
    • Simplicidad
    • Visibilidad
  • Crecimiento vertical
  • Crecimiento Horizontal
  • Usar nombres en vez de verbos
  • API
    • Put
    • Post
  • Usar HATEOAS
  • Filtros
  • Ordenamiento
  • Selección de los campos
  • Paginación
  • Versionar el API
  • Manejo de errores

    • 2xx - Éxito
    • 3xx - Redirección

    • 5xx -Error en el servidor
